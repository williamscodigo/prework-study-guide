Prework Study Guide

HTML: defines the semantic structure of a website.

  • The head element contains information about the webpage.
  • The body element contains the visible part shown to the user.
  • The header element usually houses the website navigation links.
  • The main element usualy houses the main content of a page.
  • The footer element usualy houses copyright information.
  • HTML Elements have a display of block or inline by default
  • Block level elements always start on a new line and always take the full width available.
  • Inline level elements does not start on a new line and only take as much width as necessary.
  • Examples of block elements are: div, p, header, main, footer, ul, li, etc.
  • Examples of inline elements are: span, a, img, button, etc.
  • Note: You cannot add a padding or margin to an inline level element.

CSS: specifies how elements are style and laid out on a website.

  • All browsers add intital default styles to all elements.
  • use a link tag inside the head element to point to external .css file using the href attribute.
  • You can select html element to style with css in a few ways including by tag name, by .class, by #id, and we also have advance selectors such as selectors that use an attribute type and parameterized value (ex. [href$='.com'])
  • You can reset default margin, padding and other rules to elements using the universal selector (*)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS rules that appear later in a css file can override rules that appear earlier.
  • CSS rules with more specificity take precedence over other rules.
  • Note: CSS doest allow single-line comments, it only allows multi-line comments (this can be source of unpredictable errors)

Git and Github: allows for development collaboration and source code management

  • git status checks the branch we are currently on, are there any local changes to stage and commit.
  • git checkout -b branch-name creates a new branch and switches to it.
  • git add . or git add -A will add all the files and changes to the staging area to be commited
  • git commit -m "commit message" will commit all the changes and files currently in the staging area.
  • git pull origin main will pull the remote repository and tell if there is anything new in it.
  • git push origin branch-name can be use to push the new branch and its changes to the remote repository
  • pull request can be created to have your changes and branch review and added to main branch (note: in a real world project this process has other developers reviewing your code and many stages are involved)

JavaScript: adds interactivity to a website.

  • use a script tag to point to an external .js file using the src attribute. Usually place at the end, before the closing body tag.
  • JavaScript has 7 data types: String, Number, Boolean, null, undefined, Symbol and Objects (special objects: Array, Function)
  • All primitive types, except null, can be tested by the typeof operator. typeof null returns 'object', so one has to use === null to test for null.
  • Conceptually, undefined indicates the absence of a value (use as part of core language defenitions, automatically assign/return in different situations), while null indicates the absence of an object (purposely assign/return by developers)
  • NaN ( 'Not a Number' ) is a special kind of number value that's typically encountered when the result of an arithmetic operation cannot be expressed as a number.
  • With BigInts you can safely store and operate on large integers even beyond the safe integer limit of Numbers datatype ( Number.MAX_SAFE_INTEGER || 2253 - 1 )
  • A BigInt is not strictly equal to a Number with the same mathematical value, but it is loosely equal.
  • There are many instances were undefined is use such as implicitly returns ( return; ), Accessing a nonexistent object property ( obj.iDontExist ), implicitly initialization ( let x; )
  • Note: Remember that primitive types are passed by value. While objects are passed by reference. ( That is when you store/assign an object to a variable or passed as an argument/parameter in a function, you varaible contains a reference to the object in memory NOT the actual object. )
