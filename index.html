Prework Study Guide

Linux Commands: can be used to interact with the operating system through the command-line interface.

  • ls list current files and directories.
  • ls -a list corrent files and directories ( show hidden files )
  • pwd print the path of current directory.
  • cd ['directory-name|path'] navigate through directories
  • cd .. move up one directory level
  • mkdir create a new directory
  • rm ['file-name'] remove a file
  • rm -r ['directory-name'] remove a directory recursively
  • rm -rf ['directory-name'] remove a directory recursively (without comfirmation)
  • cp ['source-file'] ['destination-file'] copy the content of one file to another file
  • cp -r ['source-directory'] ['destination-directory'] recursively copy the content of one directory to another directory
  • mv ['source-file'] ['destination-file'] move or rename files or directories
  • touch ['file-name'] create a new file
  • cat ['file-name'] show the content of a file
  • nano ['file-name'] open or create a file using the nano text editor ( note: code ['file-name'] will open the file using vs code text editor - new window )

HTML: defines the semantic structure of a website.

  • The head element contains information about the webpage.
  • The body element contains the visible part shown to the user.
  • The header element usually houses the website navigation links.
  • The main element usualy houses the main content of a page.
  • The footer element usualy houses copyright information.
  • HTML Elements have a display of block or inline by default
  • Block level elements always start on a new line and always take the full width available.
  • Inline level elements does not start on a new line and only take as much width as necessary.
  • Examples of block elements are: div, p, header, main, footer, ul, li, etc.
  • Examples of inline elements are: span, a, img, button, etc.
  • Note: You cannot set a height or width to an inline level element. (unless you change it to display: inline-block also, padding and margin for top and bottom is ignore )

CSS: specifies how elements are style and laid out on a website.

  • All browsers add intital default styles to elements.
  • use a link tag inside the head element to point to external .css file using the href attribute.
  • You can select html element to style with css in a few ways including by tag name, by .class, by #id, and we also have advance selectors such as selectors that use an attribute type and parameterized value (ex. [href$='.com'])
  • You can reset default margin, padding and other rules to elements using the universal selector (*)
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element (space between the element's content and the element's border.)
  • CSS rules that appear later in a css file can override rules that appear earlier.
  • CSS rules with more specificity take precedence over other rules.
  • Note: CSS doest allow single-line comments, it only allows multi-line comments (this can be source of unpredictable errors)
  • Note: You cannot set a height or width to an inline level element. (unless you change it to display: inline-block also, padding and margin for top and bottom is ignore )

Git and Github: allows for development collaboration and source code management

  • git config --global user.name 'your-name' set the name that will be attached to your commits
  • git config --global user.email 'your-email' set the email that will be attached to your commits
  • git config --get user.name show setup user name
  • git config --get user.email show setup user email
  • git config --global init.defualtBranch main this will make any new local repositories initialized in this computer come with a root repository of 'main' name for standard default main branch.
  • git init 'project-name' initialized/creates a new local repository in the current directory/folder
  • git clone 'project-url' clone a project from a remote repository
  • git status checks the branch we are currently on, are there any local changes to stage and commit.
  • git checkout -strong 'branch-name' creates a new branch and switches to it. better to do 1 thing at a time
  • git branch 'branch-name' [optionally: ref-branch-name] will create a new branch
  • git checkout 'branch-name' will switch/move working directory to specify branch
  • git branch -d 'branch-name' will delete the specify branch.
  • git add . or git add -A will add all the files and changes to the staging area to be commited
  • git commit -m 'commit message' will commit all the changes and files currently in the staging area.
  • git remote add origin 'remote-repository-address' will connect your local repository to the remote (github) repository
  • git pull ['remote'|'origin main'] will pull the remote repository and merge curent branch with its upstream.
  • git fetch ['remote'] will fetch changes from the remote repository but will not update tracking branches
  • git fetch --prune ['remote'] will Locally remove ref/branches that were remove from the remote repository ( example: after a pull request you get the option to safely remove that branch e.g feature/feature-name )
  • git push -u origin main can be use to push changes and -u flag setup the upstream. with upstream setup we can simple use 'git push' to push our changes from that branch. note: upstream also use by git pull
  • git push origin 'branch-name' can be use to push the new branch and its changes to the remote repository
  • git merge 'branch-name' will join specified branch into your current branch
  • A pull request can be created to have your changes and branch review and added to main branch (note: As a junior developer in a real world project this process has other developers (senior) reviewing your code and giving feedback)

JavaScript: adds interactivity to a website.

  • use a script tag to point to an external .js file using the src attribute. Usually place at the end, before the closing body tag.
  • JavaScript has 7 data types: String, Number, Boolean, null, undefined, Symbol and Objects (special objects: Array, Function)
  • All primitive types, except null, can be tested by the typeof operator. typeof null returns 'object', so one has to use === null to test for null.
  • Conceptually, undefined indicates the absence of a value (use as part of core language defenitions, automatically assign/return in different situations), while null indicates the absence of an object (purposely assign/return by developers)
  • NaN ( 'Not a Number' ) is a special kind of number value that's typically encountered when the result of an arithmetic operation cannot be expressed as a number.
  • With BigInts you can safely store and operate on large integers even beyond the safe integer limit of Numbers datatype ( Number.MAX_SAFE_INTEGER || 2253 - 1 ) (note: Number datatype also have a min for integers and a min and max for floating point values)(BigInt numbers usually have an 'n' at the end like 1000n or 0n)
  • A BigInt is not strictly equal to a Number with the same mathematical value, but it is loosely equal.
  • There are many instances were undefined is use such as implicitly returns ( return; ), Accessing a nonexistent object property ( obj.iDontExist ), implicitly initialization ( let x; )
  • Note: Remember that primitive types are passed by value. While objects are passed by reference. ( That is when you store/assign an object to a variable or passed as an argument/parameter in a function, that varaible contains a reference to the object in memory NOT the actual object. )
  • JavaScript uses type coercion in Boolean contexts. falsy values include (false, 0, -0, NaN, "", undefined, null). All other values are thruty (Example: true, any positive or negative number not equal to 0 like 5, -123, Infinity, etc, a string with 1 or more characters including "0", all objects. )
  • You can use console.log() to log code/messages to the console for debugging purposes, to view this log messages you must open the console tab in Chrome DevTools.
